Description

The sequencer has imperfections. Sometimes, it can sequence the same part twice or even more. As a result, a FASTQ file may contain several reads with the same nucleotide order. Of course, it makes it harder to assemble a puzzle when you have identical details. In this light, it is necessary to catch the repeated reads to clear the data for further processing.

Repeated reads can appear as a result of the bacterial natural repeat sequencing, but for the sake of simplicity, we do not take this feature into account.

Finding repeats is an easy task, so we decided to give you an archive with the real data. The archive contains two FASTQ files as a result of paired-end sequencing (forward reads and reverse reads). The peculiarity is the archive format â€” .sra. To extract a FASTQ file from an .sra archive, install the SRA toolkit (you can do it with Conda). The fastq dump command can help you to extract a FASTQ file. Like any other tool, the fastq dump has documentation with a list of all the options you may need. Always read it before doing something :)

These are just our recommendations; feel free to experiment and find your way to extract the file.

Objectives

  • Get the .sra archive from the link and place it in the data/ folder;
  • Extract the FASTQ files. Use only the FASTQ file with the _1.fastq postfix;
  • Find all repeated reads and print their total number. Repeats are the extra reads. For example, if the sequence of the same read occurs three times in the file, then the number of repeats is two.

[ALERT-warning]Your final program should input the path to theFASTQ file with data and print the information about it (see the Examples section).[/ALERT]
